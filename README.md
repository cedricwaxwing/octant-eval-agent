## Octant Eval Agent

An experimental agent for the Synthesis × Octant hackathon. It pulls complete historical data from the live Octant protocol and lets you ask natural language questions about epochs, projects, donors, and rewards.

The core idea: **pre‑compute a rich local dataset from Octant’s public API, then use an LLM to answer questions grounded strictly in that data.**

---

## Project structure

- `collect_data.py` – fetches all finalized Octant epochs from the production API and writes `octant_data.json`.
- `octant_data.json` – full local dataset (ignored by git; generated by `collect_data.py`).
- `agent.py` – CLI agent that:
  - loads `octant_data.json`
  - builds a targeted context slice per question (epochs + related projects)
  - calls Anthropic’s Messages API
  - returns a natural‑language answer with explicit epoch/project citations.
- `.env` – API keys and model config (never committed).
- `.env.sample` – template for `.env`.

---

## Prerequisites

- Python 3.9+ (tested with 3.9)
- Anthropic API key with access to `claude-sonnet-4-6` (or another compatible model)

---

## Setup

Clone the repo and create a virtual environment:

```bash
git clone https://github.com/<YOUR_GITHUB_USERNAME>/octant-eval-agent.git
cd octant-eval-agent

python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install requests
```

Copy the env template and fill in your keys:

```bash
cp .env.sample .env
```

Edit `.env`:

```env
ANTHROPIC_API_KEY=sk-ant-...
ANTHROPIC_MODEL=claude-sonnet-4-6
OSO_API_KEY=your_oso_key_here   # optional, reserved for future OSO integration
```

> The agent only needs `ANTHROPIC_API_KEY` and a valid `ANTHROPIC_MODEL` to run. `OSO_API_KEY` is included for future extensions (e.g. pulling live OSO metrics).

---

## Step 1 – Collect Octant data

Run the data collector once to build your local dataset:

```bash
python collect_data.py
```

This will:

- Hit the live Octant mainnet API
- Iterate over all finalized epochs
- Fetch stats, rewards, donors, patrons, allocations, and project metadata
- Save everything into `octant_data.json` in the project root

You should see progress logs per epoch and a final “Saved to octant_data.json” message.

---

## Step 2 – Run the agent (CLI)

With `octant_data.json` present and your `.env` configured:

```bash
python agent.py
```

You can now ask questions like:

- “Which projects received the highest matched rewards in epoch 4?”
- “How many donors and patrons participated in epoch 7?”
- “How did rewards for Giveth change over time?”

The agent will:

1. Parse your question for relevant epochs and project names.
2. Build a compact context from `octant_data.json` (epoch summaries plus project‑level reward traces).
3. Call the Anthropic Messages API with that context and your question.
4. Return a grounded answer that explicitly cites which epochs/projects each number comes from.

Type `quit` or `exit` to leave the CLI.

---

## Design notes

- **Local, precomputed dataset:** All Octant data is fetched once and stored in `octant_data.json`, so the agent can respond quickly without hammering the on‑chain API.
- **Structured retrieval, not raw RAG:** Instead of dumping the entire JSON into a prompt, `agent.py`:
  - indexes projects across epochs
  - detects epochs mentioned in the question
  - pulls only the most relevant stats and reward rows into the prompt.
- **Strict grounding:** The system prompt instructs the model to:
  - only use the provided data
  - explicitly mention epochs/projects when citing numbers
  - say “not in the data” rather than guessing.
